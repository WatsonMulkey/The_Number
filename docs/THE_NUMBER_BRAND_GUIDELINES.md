## Brand Identity & Positioning

### The Relationship: FOIL Industries ‚Üî The Number

**FOIL Industries** creates tools that make you better by solving complex problems with simple solutions. **The Number** is the purest expression of this philosophy applied to personal finance.

```
FOIL Industries = Sophisticated industrial workshop
The Number = The one perfect tool from that workshop
```

**Analogy**: If FOIL Industries is a master craftsman's workshop filled with precision instruments, The Number is the single, perfectly balanced tool you grab every morning‚Äîso intuitive it becomes an extension of your hand.

### Brand Positioning Statement

**The Number** is a budgeting companion for people who find traditional budgeting overwhelming or invasive. It's the anti-spreadsheet‚Äîgiving you exactly what you need (one daily spending number) without the guilt, complexity, or surveillance of conventional money apps.

**For**: Adults (25-45) who want financial control without financial obsession
**The Number is**: A daily financial compass, not a budget prison
**That**: Delivers clarity and calm through radical simplicity
**Unlike**: Mint, YNAB, EveryDollar (exhaustive tracking, category overload)
**The Number**: Respects your privacy, your time, and your intelligence

### Brand Sentiments Hierarchy

**PRIMARY** (The Number's core personality):
1. **Calm** ‚Äî Budgeting shouldn't spike your cortisol
2. **Clear** ‚Äî One number, zero confusion
3. **Respectful** ‚Äî Your data, your rules

**SECONDARY** (Inherited from FOIL):
4. **Smart** ‚Äî Sophisticated math, simple interface
5. **Honest** ‚Äî No dark patterns, no upsells
6. **In Your Corner** ‚Äî A tool that wants you to win

**Aesthetic Keywords**: Spacious, legible, breathing room, morning light, clarity, confidence

### Accessibility Requirements

**Minimum Contrast Ratios** (WCAG 2.1 AA Compliance):
- Body text: 4.5:1 (Charcoal #2B2E33 on Warm White #FAFAF8 = 12.6:1 ‚úì)
- Large text (18pt+): 3:1
- Interactive elements: 3:1

**Test Combinations**:
- ‚úì Charcoal on Warm White: 12.6:1 (excellent)
- ‚úì Slate Blue on Warm White: 5.8:1 (good for UI elements)
- ‚úó Sage Green on Warm White: 1.3:1 (decorative only, never text)
- ‚úì Mustard on Charcoal: 4.9:1 (acceptable for large text)

**Color Blind Considerations**:
Never rely on Sage Green alone to convey information. Pair with:
- Icons (checkmark for success)
- Position (top = good, bottom = needs attention)
- Text labels

## Typography

### Philosophy: Warm Intelligence

The Number's typography balances **human warmth** (serifs) with **crisp clarity** (sans-serifs). Where FOIL Industries uses the geometric precision of Space Grotesk throughout, The Number softens the edges with a serif for moments of trust and connection.

### Primary Typeface: Scope One

**Scope One** (Google Fonts: Serif, 400 weight only)

```
ABCDEFGHIJKLMNOPQRSTUVWXYZ
abcdefghijklmnopqrstuvwxyz
0123456789 $‚Ç¨¬£¬•

WHY THIS CHOICE:
‚úì High x-height for mobile legibility
‚úì Distinctive personality (not generic fintech)
‚úì Single weight keeps file size minimal (PWA performance)
‚úì Warm, slightly bookish‚Äîevokes ledgers and trust
‚úì Differentiates from FOIL's modernist Space Grotesk
```

**USE FOR**:
- **"The Number" itself** (the daily budget display): 48-72pt, Charcoal
- Onboarding headlines: 28-36pt
- Feature explanations: 18-22pt
- Any moment where warmth > efficiency

**PAIRING LOGIC**:
Scope One says "a person made this for you."
Space Grotesk (used in FOIL lockup) says "this was engineered to work."
Together = trustworthy precision with a human heart.

### Secondary Typeface: Inter

**Inter** (Google Fonts: Sans-serif, weights 400, 500, 600)

```
ABCDEFGHIJKLMNOPQRSTUVWXYZ
abcdefghijklmnopqrstuvwxyz
0123456789

WHY THIS CHOICE:
‚úì Designed for screen clarity (OpenType features for legibility)
‚úì Tabular figures (numbers align vertically in lists)
‚úì Neutral enough to let Scope One be the star
‚úì Extensive weights for hierarchy without new font files
‚úì Similar to Space Grotesk (both geometric sans) but more utilitarian
```

**USE FOR**:
- UI labels and buttons: Inter 500 (medium), 14-16pt
- Body text, transaction lists: Inter 400 (regular), 16pt
- Input fields: Inter 400, 18pt (larger for accuracy)
- Navigation: Inter 600 (semibold), 12pt uppercase

### Special Typography: Numbers

**Financial Figures** (amounts, currency):
- **Font**: Inter (tabular figures enabled)
- **Weight**: 500 for amounts under $1,000; 600 for larger
- **Size**: 20pt minimum for primary amounts
- **Color**: Charcoal (never red/green for amounts‚Äîcolor blind accessible)
- **Format**: `$1,234.56` (comma separators, always 2 decimals)

**Currency Symbol Styling**:
- Same size as figures (don't superscript)
- Same weight
- Position: Leading (`$50.00` not `50.00$`)

### Font Loading Strategy (PWA Performance)

**Priority 1 (Inline Critical CSS)**:
```css
/* Load immediately */
@import url('https://fonts.googleapis.com/css2?family=Scope+One&display=swap');
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600&display=swap');
```

**Fallback Stack**:
```css
font-family: 'Scope One', Georgia, 'Times New Roman', serif;
font-family: 'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
```

**Loading Behavior**:
- Show fallback immediately (no FOIT‚Äîflash of invisible text)
- Swap to web font when loaded (acceptable brief reflow for brand consistency)

## Visual Language

### Design Principles

**1. One Thing at a Time**
Each screen has a clear primary action. No competing CTAs. The eye should know where to land instantly.

**2. Generous Space is a Feature**
White space isn't wasted‚Äîit's the pause between thoughts. Aim for 60-70% negative space on any screen.

**3. Progressive Disclosure**
Show the number. Hide the complexity. Advanced features exist behind a tap, never on the main screen.

**4. Calm Interactions**
No confetti, no sudden animations. Transitions should feel like breathing: smooth, natural, unforced.

### Layout System

**Grid Foundation** (Mobile-First):
```
Container: 100% width, 16px horizontal padding
Maximum width: 480px (center on larger screens)
Vertical rhythm: 8px base unit (all spacing is multiples of 8)

Spacing Scale:
- xs: 8px (tight element spacing)
- sm: 16px (default component padding)
- md: 24px (between sections)
- lg: 40px (major section breaks)
- xl: 64px (page-level separation)
```

**The Number Display** (Hero Element):
```
Position: Top third of screen, horizontally centered
Background: Sage Green #E9F5DB panel
Padding: 40px vertical, 24px horizontal
Border Radius: 16px (soft, approachable)
Shadow: 0 4px 12px rgba(43, 46, 51, 0.08) (subtle lift)

Typography:
"You have" ‚Äî Inter 400, 16pt, Charcoal at 70%
"$127.50" ‚Äî Scope One 400, 72pt, Charcoal
"to spend today" ‚Äî Inter 400, 16pt, Charcoal at 70%
```

**Card Style** (Transactions, Settings Panels):
```

Border Radius: 12px
Padding: 16px
Shadow: None (flat, let content hierarchy create depth)
Hover/Active: Border ‚Üí Slate Blue #4A5F7A
```

### Motion & Animation

**Philosophy**: Calm, purposeful, never gratuitous.

**Micro-interactions**:
```
Button Press:
- Scale: 0.98 (subtle shrink)
- Duration: 100ms
- Easing: ease-out

Screen Transitions:
- Type: Slide (right to left for forward, left to right for back)
- Duration: 250ms
- Easing: cubic-bezier(0.4, 0, 0.2, 1) (Material Design standard)

Number Update (daily refresh):
- Type: Fade out old ‚Üí Fade in new
- Duration: 400ms
- Easing: ease-in-out
- Avoid: Flip counters, slot machines (too gamified)
```

**Loading States**:
- **Skeleton Screens**: Gray placeholder shapes (not spinners)
- **Color**: Soft Gray #D4D8DC with 1s pulse animation
- **Copy**: "Calculating your number..." (Inter 400, 16pt, Charcoal at 50%)

## Tone & Voice

### The Number's Personality

If The Number were a person, they'd be the friend who tells you the truth kindly, celebrates your wins without judgment, and never makes you feel bad about a setback.

**Core Attributes**:
- **Honest**: No sugar-coating, but no shame either
- **Encouraging**: Assumes you're trying your best
- **Concise**: Respects your time (3-sentence max for any explanation)
- **Grounded**: No hype, no financial jargon

**NOT**:
- ‚ùå Gamified ("You leveled up! üéâ")
- ‚ùå Parental ("You shouldn't have spent that")
- ‚ùå Guru-ish ("Here's the secret to wealth...")
- ‚ùå Corporate ("Optimize your financial wellness journey")

### Voice Guidelines

**FOIL Industries Voice** (for comparison):
- Authoritative, confident, industrial
- "We engineer solutions to complex problems"
- Speaks to makers, doers, professionals

**The Number Voice**:
- Conversational, calm, present-tense
- "Here's what you can spend today"
- Speaks to anyone who's tired of budgeting guilt

**Sentence Structure**:
- **Short sentences**: 8-15 words average
- **Active voice**: "You have $127 left" not "There is $127 remaining"
- **Direct address**: "You," "Your," never "The user"
- **Contractions**: "You're on track" not "You are on track" (warmer)

```
‚úì "Budgeting, simplified. We'll calculate one number: how much you can spend today."
‚úó "Welcome to The Number! Get ready to transform your financial life with our revolutionary approach!"

### Word Choice Glossary

**Use** ‚Üí **Avoid**
- "Your number" ‚Üí "Your budget" (softer, less restrictive)
- "Expenses" ‚Üí "Bills" (broader, less negative)
- "Transactions" ‚Üí "Spending events" (simpler)
- "Update" ‚Üí "Modify" / "Edit" (friendlier)
- "You're over" ‚Üí "Deficit" / "Overdrawn" (less financial jargon)
- "Add" ‚Üí "Input" / "Enter" (more human)

### Punctuation & Formatting

**Periods**: Use for complete sentences. Skip for short labels ("Add transaction" not "Add transaction.")

**Exclamation Points**: Maximum one per screen. Reserve for genuine wins only.

**Question Marks**: Avoid rhetorical questions ("Ready to save money?"). Use for actual choices ("Delete this expense?")

**Numbers**:
- Spell out one through nine ("three expenses")
- Numerals for 10+ ("15 transactions")
- Always numerals for currency ("$5" not "five dollars")

**Capitalization**:
- Sentence case for all UI (not Title Case)
- Proper nouns: "The Number," "FOIL Industries"
- Never ALL CAPS for emphasis (use bold)

```

Tablet: 768px - 1023px
Desktop (Rare): 1024px+

- Primary CTA: 48px height minimum

**Thumb-Friendly Zones**:
- Place primary actions in bottom third of screen (natural thumb reach)
- Top third: Display info only (The Number, context)
- Middle third: Scrollable content (transactions, settings)

## Accessibility Standards

### Commitment

The Number is built for everyone. Accessibility isn't a checklist‚Äîit's about respecting every user's ability to manage their money with dignity.

### Visual Accessibility

**Color Contrast** (WCAG 2.1 Level AA):
- ‚úì All text: 4.5:1 minimum (7:1 for Level AAA where possible)
- ‚úì UI components: 3:1 minimum
- ‚úì Tested combinations listed in Color System section

**Color Independence**:
- Never use color alone to convey meaning
- ‚úì "Over budget": Show amount in Terracotta + up arrow icon
- ‚úó "Over budget": Show amount in red only

**Typography**:
- Minimum size: 16px body text (exception: 14px for metadata if 60%+ contrast)
- No italics for body text (harder to read, especially for dyslexia)
- Line height: 1.5-1.6 (generous spacing aids comprehension)
- Line length: 65 characters maximum

**Focus States**:
```css
Interactive elements must have visible focus:
- Outline: 2px solid Slate Blue #4A5F7A
- Offset: 2px from element
- Contrast: 3:1 against background
- Never: outline: none; (unless custom focus style provided)
```

### Motor Accessibility

**Touch Targets**:
- 48x48px minimum (exceeds WCAG 2.5.5)
- Spacing: 8px between targets

**Gestures**:
- No swipe-only interactions (provide button alternative)
- No multi-finger gestures (pinch, two-finger scroll)
- Timeout: None (user controls pace)

### Cognitive Accessibility

**Simplicity**:
- One primary action per screen
- Consistent patterns (same icon = same action throughout)
- Undo available for destructive actions

**Language**:
- 8th-grade reading level or below
- Active voice, short sentences
- Avoid idioms / financial jargon

**Predictability**:
- Navigation always in same location
- Back button works as expected (browser history)
- No auto-play animations (user triggered only)

### Screen Reader Support

**Semantic HTML**:
```html
<header>, <nav>, <main>, <footer> (not <div class="header">)
<button> for actions (not <div onclick="">)
<input type="number"> for amounts (not text)
```

**ARIA Labels**:
```html
<!-- The Number display -->
<div role="region" aria-label="Today's available spending amount">
  <p aria-hidden="true">You have</p>
  <p class="the-number" aria-label="$127.50">$127.50</p>
  <p aria-hidden="true">to spend today</p>
</div>

<!-- Icon buttons -->
<button aria-label="Add new transaction">
  <svg aria-hidden="true"><!-- plus icon --></svg>
</button>
```

**Alternative Text**:
- Decorative images: `aria-hidden="true"` or `alt=""`
- Informative icons: Descriptive alt text
- Illustrations: Convey meaning, not description (alt="No transactions added yet" not "Drawing of a hand holding a coin")

### Testing Checklist

**Before Launch**:
- [ ] Run axe DevTools (automated accessibility testing)
- [ ] Test with VoiceOver (iOS) and TalkBack (Android)
- [ ] Navigate entire app using keyboard only (Tab, Enter, Esc)
- [ ] Test with 200% browser zoom (text should reflow)
- [ ] Simulate color blindness (Deuteranopia, Protanopia, Tritanopia)
- [ ] Test in bright sunlight (outdoor contrast check)
- [ ] User test with at least one person with disabilities
